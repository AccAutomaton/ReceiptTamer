# 报销导出依据高保真确认稿

> APPROVED · MIGRATED TO FLUTTER

本稿已迁移到 Flutter 报销一级页面，作为双依据选择结构的验收基准：

- 默认“按订单导出”，进入报销页即展示订单选择清单。
- “按订单导出 / 按发票导出”作为标题右侧的纯文字下拉选项，不再使用独立中转页；控件背景与边框透明，内容居中并使用正文墨色。
- 下拉弹层固定屏幕水平居中，使用紧凑的双行纸签选项；主标题说明导出依据，副标题说明同票联动方式，选中态仅以薄荷底色和勾标识，不使用默认灰色菜单块。
- 保留老版全选、反选、日期筛选、同票联动、不可选状态、金额统计和下一步导出选项；同票联动以状态栏下方的低饱和扁平纸签反馈，数字只计算本次操作实际一并选中或取消的订单。该纸签同时作为全应用短时反馈的视觉基准，成功、警告、错误和信息提示均从顶部出现并保持相同纸色、描边、圆角与动效。
- 两种依据共用连续账页和固定选择栏；中间列表在顶栏及固定选择栏处使用雾边。
- 账页日期栏不单独着色，始终继承整行的选中或未选底色，只保留细分隔线。
- 报销勾选账页在勾选栏与日期栏之间补充连续细分隔线，普通无勾选账页不重复绘制左边线。
- “导出记录”保留为一级页面的轻量入口，不占用清单空间。

三个手机状态依次为：默认按订单、下拉展开、切换到按发票。

实现位于 `lib/presentation/screens/export/reimbursement_screen.dart`，订单与发票选择状态分别位于 `export_provider.dart` 和 `invoice_export_provider.dart`；行为由 `test/reimbursement_ui_flow_test.dart` 与 UI 源码契约持续约束。
