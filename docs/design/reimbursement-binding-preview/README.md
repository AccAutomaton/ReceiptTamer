# ReceiptTamer「报销装订轨」已确认视觉与功能基准

> APPROVED · MIGRATED TO FLUTTER

这组画板是 Flutter 信息架构迁移的基准。迁移已退役 `ce04b3c` 页面哈希冻结，同时保持现有持久化模型、导入/识别/关联/导出能力与晨雾品牌连续性；迁移后的细节修订以本文、真实 Flutter、Widget 测试和 Golden 为准。

## 核心判断

- 全套画板改用衬线体系：中文使用 Noto Serif SC，英文与数字使用 Tinos；标题、正文、标签、金额和导航不再混用无衬线体。
- 所有应用内按钮、卡片、输入字段、导航、Sheet 与固定操作栏改用扁平表面：仅以实体填充、细描边和状态色表达层级，移除厚底边、内高光、按压下沉与实体阴影。手机外框和总览画板仍可保留用于展示边界的外部阴影。
- 首页成为全局资料与高频任务的分流台，不承载报销周期，也不把“未关联发票”一律判定为待办或错误。
- 首页由两张独立纸卡组成：上方是仅保留三点装订的功能目录，其中开票助手为整行主入口，内部以柔和印章图标、标题及两行强调数字、“查看未关联订单”与行动箭头组成和统计区对齐的三栏；栏间使用短票据虚线，不再形成三格表格。用餐证明导出与发票导出为并列小卡，报销材料只从主导航进入，不在首页重复。下方“最近订单”只展示按收录时间排序的最新 10 条订单，“查看更多”切换到订单主导航并同步底栏选中态。两张卡留出 12–14px 间隔并共用一个外层滚动区，不产生嵌套滚动；滚动视口在固定导航上方截止，以背景渐入/内容渐隐提示仍有资料。
- 所有以列表或连续字段组作为中间内容的页面统一使用“雾边”：内容进入顶栏下方时由背景中淡入，接近底部导航、操作栏或批选栏时渐隐。渐隐层不拦截触控，结束线与真实滚动边界一致；没有固定底控件的设置页只保留顶部效果。
- 订单和发票主列表改为按月连续账页：月份抬头与记录属于同一张纸，每行仅保留日期边栏和细分隔线，不再重复订单/发票图标、圆角卡片、阴影和状态胶囊。每个月份账页的四个外角统一为 16dp，内部行仍连续相接。两个列表都支持月份与关联状态筛选，再次点选已选条件或使用“清除筛选”即可恢复全部；月份抬头在本月数据可见期间组内粘性停靠，总结内容上下各保留 28dp 雾边安全区，前景描边保持圆角完整。快速月份轨仅占用半个命中宽度，账页与轨道之间保留 4dp 间隔，避免切角或形成重复右边距。
- “报销”进入即展示可选择的连续月份账页，默认按订单导出，也可从标题区域居中透明下拉切为按发票导出；日期只是可选筛选，最终范围由本次勾选决定。
- 底部主导航改为连续的“首页—订单—发票—报销”；“新增”是与导航同高的右下角独立侧键。低频“关于”与模型、存储、备份一起移入“设置”。顶栏搜索、筛选、设置等图标按钮使用透明背景与透明边框，设置列表增加行间距；更新历史的重试动作不显示无关的加号。
- 关联支持两个方向：开票助手从未关联订单出发，按店铺批选并在保存新发票时自动关联；发票录入则从发票出发，根据日期和金额建议订单，只建议、由用户确认后写入关系。
- “报销”页以显式订单与发票 ID 进入完整三件套生成，并保留轻量“导出记录”入口；只导出用餐证明则从首页快捷路径直接选择订单，不受发票关系限制。

## 首版语义与实现边界

- 报销范围只在用户发起导出时通过勾选订单或发票形成；可用任意起止日期过滤清单，但日期不等于导出范围，也不默认自然月，不在首页、订单、发票或收件流程创建“周期”语义。
- 首版不持久化“报销批次”；两种依据的选择只作为本次生成的进程内输入。
- “未关联”只表示订单在应用内尚无发票关系，不判断税务平台是否已经开票；开票助手负责新建发票并关联，关联已有发票仍从订单详情进入。
- 用餐证明以用户勾选的订单为范围；日期只是该页面内可选的订单筛选，默认全部日期，没有发票的订单也可以生成证明。
- 开票助手默认查看全部日期的未关联订单，按店铺聚合；一次只选择一个店铺，保存新发票时复用预选订单并写入关联，不要求用户二次选择。
- “可导出”只表示所选订单与发票在应用内具有关联关系，不表示材料已提交或报销已审批。
- AI 画板只表达字段“已填 / 需确认”，不承诺当前 OCR/LLM 返回总置信度或字段置信度。
- 分享导入队列只在当前会话内存在；退出或重启后不承诺继续。图片用途仍由用户选择，系统不根据文件外观自动判定订单或发票。
- 文本型 PDF 可进入字段提取；扫描型或图片型 PDF 可能需要手工填写。
- 按订单导出时，选择任一订单会自动带入同一张发票下的其他订单，包括日期筛选范围外的关联订单；按发票导出时直接带入该发票的全部订单。未关联记录不可选。该规则不适用于首页的独立用餐证明导出。
- 已导出文件从 `Download/ReceiptTamer` 读取，按生成日期展示，无法从旧文件可靠反推报销月份，也不属于当前应用备份与还原内容。
- 画板中的“分享全部”是新增的多文件分享提案，需要补充 Android 多文件分享能力；逐个文件分享可直接复用现有能力。
- Flutter 已注册 Tinos 四字形并以 Noto Serif SC 回退中文；360dp、2.0× 字号、中文长店名和暗色小字号由 Widget 测试与 Golden 持续约束。

以上首版边界不需要新增持久化字段，不改变数据库版本、备份包格式、还原逻辑或应用版本号。若未来保存报销批次、人工核对状态或跨启动导入草稿，则必须重新审视数据库迁移与备份/还原。

    pnpm run render:design:binding
